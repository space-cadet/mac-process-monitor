## Details

Built a lightweight web dashboard showing live battery, CPU, memory, disk I/O, network, and process data. Serves from a standalone TypeScript process using native Node.js `http` module.

## API Endpoints
| Endpoint | Description |
|---|---|
| `GET /api/snapshots?minutes=60` | Recent system snapshots (newest first) |
| `GET /api/processes?limit=20` | Top processes from latest snapshot |
| `GET /api/drain-events` | All drain events with top processes |
| `GET /api/stats` | DB stats (snapshot count, event count) |

## Decisions
- **Native Node.js http** — No Express dependency, keeps project lightweight
- **Chart.js from CDN** — No build step for frontend, always latest
- **Port 3456** — Consistent with chimera-chat demo
- **Dark theme** — `#0f172a` background, card-based layout
- **Standalone process** — Dashboard can run independently of monitor (`npx tsx src/dashboard.ts`)
- **No auth** — Local-only, assumed firewall/trust boundary

## Known Issues
- IO charts show cumulative counters (very large numbers), not rates — needs delta calculation
- Battery chart is irrelevant on desktop/VPS (always 0%)
- Dashboard server does not auto-start with monitor (separate process by design)
- Process table is server-side sort only (no client-side column sorting)
