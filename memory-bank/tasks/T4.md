## Overview

A fully-featured web dashboard for real-time monitoring of macOS system processes, battery drain, and resource spikes. Built with native Node.js HTTP (no Express), Chart.js, and a modular 7-file frontend architecture.

## Dashboard Architecture

### Backend (`src/dashboard/server.ts`)
- Native Node.js `http` module — zero dependencies beyond project stack
- Serves static files from `public/` with proper MIME types
- 12 JSON API endpoints for all data queries

### Frontend (7 modular files, 400–600 lines total)
| File | Purpose | Size |
|------|---------|------|
| `index.html` | Layout structure, side-by-side grid, modals | ~200 lines |
| `style.css` | Dark theme, responsive breakpoints, cards, modals | ~430 lines |
| `utils.js` | Formatting, sorting, DOM helpers, API wrappers | ~120 lines |
| `charts.js` | Chart.js initialization, 5 chart instances | ~150 lines |
| `tables.js` | Sortable table, process modal, spike/battery panels | ~170 lines |
| `profiles.js` | Profile CRUD UI, color picker, form modal | ~90 lines |
| `app.js` | Orchestration: data fetch, refresh loop, events | ~70 lines |

## API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/snapshots` | GET | Historical system snapshots (last N minutes) |
| `/api/processes` | GET | Latest process list (top N by CPU) |
| `/api/drain-events` | GET | Battery drain events with top process correlation |
| `/api/stats` | GET | DB snapshot count, event count, size in bytes |
| `/api/spikes` | GET | Recent resource usage spikes (CPU/memory) |
| `/api/spike-stats` | GET | Aggregated spike statistics |
| `/api/battery-impact` | GET | Ranked battery impact scores per process |
| `/api/battery-events` | GET | Individual drain period details |
| `/api/process-history` | GET | Per-process timeline (CPU + memory over time) |
| `/api/process-stats` | GET | Per-process statistics (avg/peak/min/max) |
| `/api/top-processes` | GET | Top N processes by CPU or memory |
| `/api/profiles` | GET/POST | Monitoring profile list / create |
| `/api/profiles/:id` | PUT/DELETE | Update / delete profile |

## UI Features

### Side-by-Side Layout
- **Left column**: Process table (sortable), spike alerts, drain events
- **Right column**: CPU/Memory chart, battery chart, load/swap/temp chart, disk/network I/O chart, battery impact rankings
- **Bottom**: Monitoring profiles grid
- **Responsive**: Stacks to single column on mobile (`< 900px`)

### Sortable Process Table
- Click any column header to sort (CPU%, Memory%, PID, Name, etc.)
- Arrow indicator shows current sort direction
- Click any row → opens process detail modal

### Process Detail Modal
- **Statistics panel**: Sample count, avg/peak CPU & memory, first/last seen timestamps
- **History chart**: 60-minute CPU + memory trend line (Chart.js)
- **Recent samples table**: Timestamped data points with CPU% and Mem%

### Spike Alert Panel
- Real-time feed of recent resource spikes (last 60 minutes)
- Shows process name, spike value, baseline, threshold
- Click any spike → opens process detail modal
- Color-coded: CPU spikes = 🔴, Memory spikes = 🟡

### Battery Impact Rankings
- Bar chart visualization of accumulated impact scores
- Shows drain time, sample count, avg CPU per process
- Click any bar → opens process detail modal

### Monitoring Profiles
- Create/edit/delete named process groups
- Each profile: name, color, list of process name patterns
- Click a profile → filters process table to show only matching processes
- Uses: group "Browsers" (Chrome + Safari + Firefox), "Dev Tools" (VSCode + node + tsx), etc.

## Database Integration
- Dashboard reads from `~/.procmon/monitor.db` (SQLite)
- Queries `snapshots`, `process_samples`, `drain_events`, `process_spikes`, `battery_impact`, `battery_impact_events` tables
- No caching — live data on every refresh

## Testing
- Live verified: all 12 API endpoints return correct data
- Frontend verified: all charts render, table sorts, modal opens, profiles CRUD works
- Auto-refresh: every 5 seconds

## Usage
```bash
# Start the monitor (collects data)
npx tsx src/main.ts

# Start the dashboard (serves UI)
npx tsx src/dashboard.ts 3456

# Open in browser
open http://localhost:3456
```
