## Description

Track battery usage over time and determine which processes deplete the battery the most. The system correlates battery drain periods with per-process CPU activity, computing an impact score that accumulates over time to identify the biggest battery drains.

## Implementation

### Files Created
- `src/core/BatteryImpactAnalyzer.ts` ã Battery drain correlation engine
  - Detects drain periods (battery dropping while not charging)
  - Aggregates per-process CPU usage during drain windows
  - Computes impact score = process's share of total CPU-seconds during drain
  - Normalizes by total battery drop to get comparable scores

## How Impact Scoring Works

1. **Drain period detection**: When battery is not charging and dropping, the analyzer starts a drain period
2. **Process aggregation**: For each sample in the drain period, accumulate CPU-seconds per process (CPU% û duration)
3. **Impact score**: Each process's share of total CPU-seconds during the drain period, scaled by the battery drop percentage
4. **Accumulation**: Scores accumulate over time via `INSERT ... ON CONFLICT DO UPDATE`
5. **Ranking**: Query `battery_impact` ordered by `total_impact_score` to see the worst offenders

## Features

- **Automatic drain detection**: No manual configuration needed; monitors battery state continuously
- **Per-process impact**: Every process gets a score based on its CPU activity during drain
- **Accumulated over time**: Long-running tracking gives accurate picture of habitual battery drains
- **CLI query**: `npx tsx src/query.ts --battery --limit 10` shows top battery drainers
- **Event history**: All drain periods stored as `battery_impact_events` for detailed analysis
