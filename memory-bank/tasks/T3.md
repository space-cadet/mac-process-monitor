## Overview

A CLI query interface for per-process history, statistics, and rankings. Accessible via `src/query.ts` with multiple query modes.

## Implementation

### Files Created/Modified
- `src/query.ts` — Main CLI query tool with multiple options
- `src/storage/TimeSeriesDB.ts` — Added query methods:
  - `getProcessHistory(processName, cutoff)` — Timeline of CPU/memory for a process
  - `getProcessStats(processName, cutoff)` — Aggregated statistics (avg/peak/min/max)
  - `getTopProcesses(metric, limit, cutoff)` — Top N processes by CPU or memory

### CLI Usage

```bash
# Show recent spikes
npx tsx src/query.ts --spikes --since 1h

# Show spike statistics (aggregated)
npx tsx src/query.ts --spikes --stats --since 1h

# Show battery impact rankings
npx tsx src/query.ts --battery --limit 10

# Show battery drain events
npx tsx src/query.ts --battery-events --limit 5

# Show general stats
npx tsx src/query.ts --stats

# Show top processes by CPU
npx tsx src/query.ts --top --metric cpu --limit 10

# Show top processes by memory
npx tsx src/query.ts --top --metric mem --limit 10

# Show aggregated process statistics
npx tsx src/query.ts --process-stats --limit 20

# Show per-process history (default: last 60 minutes)
npx tsx src/query.ts --process node --since 1h

# Query drain events
npx tsx src/query.ts --drain-events --limit 5
```

## API Endpoints (Dashboard Integration)

The same queries are exposed via the web dashboard API:

| Endpoint | Parameters | Description |
|----------|------------|-------------|
| `/api/process-history` | `process`, `minutes` | Timeline of CPU/memory samples |
| `/api/process-stats` | `process`, `minutes` | Aggregated statistics |
| `/api/top-processes` | `metric` (cpu/mem), `limit`, `minutes` | Top N processes |
| `/api/spikes` | `minutes` | Recent spike alerts |
| `/api/spike-stats` | `minutes` | Aggregated spike data |
| `/api/battery-impact` | `limit` | Ranked battery impact |
| `/api/battery-events` | `limit` | Individual drain events |

## Features

- **Time-range filtering**: `--since 1h`, `--since 24h`, `--since 7d`
- **Metric selection**: CPU or memory for top-process queries
- **Limit control**: Configurable number of results
- **Stats aggregation**: Min/max/avg/peak calculations per process
- **Dashboard integration**: All CLI queries available as HTTP API endpoints
