# T6: Process Resource Usage Spike Detection

## Status: ‚úÖ Completed

## Description

Implement a spike detection system that tracks individual processes and identifies/logs resource usage spikes. The system maintains per-process baselines (moving averages) and detects when a process exceeds either absolute thresholds or relative multipliers above its baseline.

## Implementation

### Files Created
- `src/core/SpikeDetector.ts` ‚Äî Core spike detection engine
  - Per-process baseline tracking (moving average of CPU and memory)
  - Dual-threshold detection: absolute (% CPU/% memory) and relative (multiplier above baseline)
  - Cooldown per process to avoid spam
  - Watchlist and ignorelist support

### Files Modified
- `src/types/index.ts` ‚Äî Added `SpikeThresholds`, `ProcessSpike`, `SpikeConfig` interfaces
- `src/storage/TimeSeriesDB.ts` ‚Äî Added `process_spikes` table and methods:
  - `insertProcessSpike()`
  - `getRecentSpikes()`
  - `getSpikeStats()`
- `src/core/Monitor.ts` ‚Äî Integrated `SpikeDetector` into the tick loop
  - Default config: CPU threshold 50%, memory threshold 20%, 3x multiplier, 5 baseline samples, 60s cooldown
  - Ignores: `kernel_task`, `WindowServer`, `mds`, `mdworker`
- `src/query.ts` ‚Äî Added `--spikes` query option with stats and timeline views

### Database Schema

```sql
CREATE TABLE process_spikes (
  id TEXT PRIMARY KEY,
  timestamp INTEGER NOT NULL,
  process_name TEXT NOT NULL,
  pid INTEGER NOT NULL,
  metric_type TEXT NOT NULL,  -- 'cpu' | 'memory'
  value REAL NOT NULL,        -- Actual spiked value
  baseline REAL NOT NULL,     -- Moving average baseline
  threshold REAL NOT NULL,    -- Threshold that was crossed
  snapshot_id INTEGER         -- Foreign key to snapshots
);
```

## Features

- **Baseline tracking**: Maintains a rolling window of CPU/memory samples per process
- **Dual threshold**: Absolute (e.g., CPU > 50%) OR relative (e.g., 3x above baseline)
- **Smart cooldown**: 1-minute cooldown between spikes for the same process
- **Filtering**: Watch only specific processes, or ignore system processes
- **CLI queries**: `npx tsx src/query.ts --spikes --since 1h` shows spike history
- **Stats view**: `npx tsx src/query.ts --spikes --stats --since 1h` shows aggregated spike stats
